That command works against your live DB. Here's the full set you asked for.
1. View all tables
docker exec clari5pay_db psql -U postgres -d clari5pay -c "\dt"
Your 7 tables: users, transactions, account_master, account_transaction, notifications, system_logs, support_messages.


2. View the data in every table (readable summary)
docker exec clari5pay_db psql -U postgres -d clari5pay -c "SELECT '--- USERS ---' AS section; SELECT id, username, role, name, email, active, merchant_role FROM users ORDER BY id; SELECT '--- TRANSACTIONS ---'; SELECT id, ref, type, amount, status, merchant_name FROM transactions ORDER BY id; SELECT '--- ACCOUNT_MASTER ---'; SELECT id, reference_number, account_name, bank_name, status FROM account_master ORDER BY id; SELECT '--- ACCOUNT_TRANSACTION ---'; SELECT * FROM account_transaction ORDER BY id; SELECT '--- NOTIFICATIONS ---'; SELECT id, user_id, message, read FROM notifications ORDER BY id; SELECT '--- SYSTEM_LOGS ---'; SELECT id, actor_name, action, detail FROM system_logs ORDER BY id DESC; SELECT '--- SUPPORT_MESSAGES ---'; SELECT id, merchant_id, sender, content FROM support_messages ORDER BY id;"
I used trimmed columns on purpose — transactions/notifications hold huge base64 image strings (admin_proof, merchant_proof) and users holds the password hash, which would flood the screen.


3. View ALL columns of ONE table (full row, easy to read)
The -x flag prints each row vertically (key: value), so long values don't wrap badly:
docker exec clari5pay_db psql -U postgres -d clari5pay -x -c "SELECT * FROM transactions ORDER BY id;"
Swap transactions for any table name.
4. Delete all data except the super admin
Runs in one atomic transaction; the WHERE role <> 'SUPER_ADMIN' guard means the super admin is never touched. Order matters (foreign keys):
docker exec clari5pay_db psql -U postgres -d clari5pay -c "DELETE FROM account_transaction; DELETE FROM transactions; DELETE FROM support_messages; DELETE FROM notifications; DELETE FROM system_logs; DELETE FROM account_master; DELETE FROM users WHERE role <> 'SUPER_ADMIN';"


This wipes all transactions, accounts, notifications, logs, support messages, and every admin/merchant — leaving only superadmin / pass123.
Heads-up from the data above: you currently have an admin merchant1 (name "priya", deactivated) and one account ACC0000001 — both will be removed by command #4. Run it whenever you want a clean slate.
